# 23：真实开源中文数据集小规模预训练实验

## 为什么这一课开始使用真实开源中文数据集？

前面的中文实验使用 toy text。

toy text 的优点是可控、干净、容易验证；缺点是太小、太重复，不像真实预训练语料。

本节开始从真实开源中文数据集中抽样，目标不是训练出高质量模型，而是理解真实数据进入模型训练前的工程流程。

---

## 为什么使用 streaming / 抽样？

真实预训练数据集通常很大。

如果直接全量下载，会浪费网络、磁盘和时间。

所以本节使用 streaming 或按需读取方式，只抽取有限样本：

```text
max_docs
max_chars
```

达到任一上限就停止。

---

## 真实数据和 toy text 的区别

真实数据更复杂：

1. 长短不一。
2. 内容来源混杂。
3. 可能有重复。
4. 可能有网页噪声。
5. 可能混入非中文文本。
6. 标点、数字、英文、链接都更多。

所以真实数据训练前必须先做清洗和统计。

---

## 数据清洗为什么重要？

清洗不是为了让文本完美，而是为了避免明显坏数据影响训练。

本节做了：

1. 空文本过滤。
2. 过短文本过滤。
3. 中文比例过滤。
4. 重复文本过滤。
5. 基础 HTML / URL / 空白符清理。

---

## 中文比例、长度过滤、去重分别解决什么问题？

中文比例过滤：避免大量英文、代码、乱码进入中文预训练。

长度过滤：去掉过短文本，因为它们缺少上下文；过长文本先截断，避免单条样本过大。

去重：减少模型重复记忆相同内容。

---

## BPE tokenizer 如何接入真实数据？

本节优先尝试复用第 22 课 tokenizer。

如果复用 tokenizer 后 `<unk>` 比例太高，就会基于本课真实 corpus 重新训练 BPE tokenizer。

这样可以避免真实数据里大量新字符被编码成未知 token。

---

## train / val split 的意义

train tokens 用来更新模型参数。

val tokens 不参与训练，只用来观察模型是否泛化。

如果 train loss 明显低于 val loss，通常说明模型开始记忆训练数据。

---

## loss 和生成效果怎么看？

loss 下降说明模型越来越能预测下一个 token。

但 loss 下降不等于生成文本一定自然。

小模型、小数据、短训练的生成结果通常会有重复、断裂或语义不稳定。

---

## 为什么这还不是真正大模型预训练？

因为本节规模仍然很小：

1. 数据只是抽样。
2. tokenizer 很小。
3. Tiny GPT 参数量很小。
4. 训练步数有限。
5. 没有分布式训练、混合数据、质量分层和现代大模型训练技巧。

---

## M1 Pro / 32GB 适合做什么规模？

适合：

1. 数据管线实验。
2. tokenizer 实验。
3. Tiny GPT 小模型训练。
4. 几百万字符级别的小样本预训练。
5. loss 曲线和生成样本观察。

不适合直接训练真正的大语言模型。
