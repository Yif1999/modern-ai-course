# 第 8 课：CNN 的直觉：为什么图片不适合直接 flatten？

## 这一课解决什么问题？

上一课我们用 MLP 做 MNIST 分类。

MLP 的做法是先把图片：

```text
[1, 28, 28]
```

展平成：

```text
[784]
```

这可以训练，但它有一个问题：图片原本的二维空间结构被打散了。

例如，一个笔画在图片左上角附近，它和周围像素之间有空间关系。flatten 之后，模型仍然能看到 784 个数字，但不再天然知道哪些像素是上下左右相邻的。

## 图片的二维空间结构

MNIST 图片原始形状是：

```text
[channel, height, width] = [1, 28, 28]
```

如果加上 batch，则是：

```text
[batch, 1, 28, 28]
```

其中：

- `batch` 表示一次训练多少张图片。
- `1` 表示灰度图只有一个通道。
- `28, 28` 表示图片的高度和宽度。

CNN 会保留这种二维结构，而不是一开始就把它展平。

## flatten 的问题

flatten 会把：

```text
[batch, 1, 28, 28]
```

变成：

```text
[batch, 784]
```

这会丢掉显式的二维位置信息。

MLP 的第一层会把每个像素和隐藏层全连接起来。它当然可以学习，但它没有利用图片里很重要的先验：

```text
相邻像素通常更相关
局部笔画比单个孤立像素更有意义
同一种笔画可能出现在图片不同位置
```

## convolution 的局部感受野

卷积层不会一次看完整张图片。

例如 `kernel_size=3` 表示每次看一个：

```text
3 x 3
```

的小窗口。

这个小窗口就是局部感受野。

对于手写数字，很多有用特征都是局部的，例如：

- 横线
- 竖线
- 斜线
- 小弯曲
- 边缘

CNN 先学习这些局部特征，再逐层组合成更高级的数字特征。

## 权重共享

卷积核会在整张图片上滑动。

同一个卷积核在不同位置使用的是同一组参数。

这叫权重共享。

它的好处是：

```text
同一种局部特征无论出现在左边、右边、上边、下边，都可以被同一个卷积核检测到。
```

这比 MLP 更适合图片。

## feature map 是什么？

卷积层输出的结果叫 feature map。

例如本节第一层卷积：

```python
Conv2d(in_channels=1, out_channels=8, kernel_size=3, padding=1)
```

输入是：

```text
[batch, 1, 28, 28]
```

输出是：

```text
[batch, 8, 28, 28]
```

意思是：

```text
模型学习了 8 个卷积核
每个卷积核生成一张 feature map
```

所以 8 个输出通道就是 8 张特征图。

## pooling 是什么？

本节使用：

```python
MaxPool2d(kernel_size=2)
```

它会把空间尺寸减半：

```text
28 x 28 -> 14 x 14
14 x 14 -> 7 x 7
```

pooling 的作用是压缩空间尺寸，同时保留比较强的局部响应。

## 本节 CNN 结构

本节模型结构是：

```text
输入: [batch, 1, 28, 28]
Conv2d: 1 -> 8
ReLU
MaxPool: 28 x 28 -> 14 x 14
Conv2d: 8 -> 16
ReLU
MaxPool: 14 x 14 -> 7 x 7
Flatten: [batch, 16, 7, 7] -> [batch, 784]
Linear: 784 -> 10
```

注意：CNN 最后仍然会 flatten。

区别是：

```text
MLP 是一开始就 flatten
CNN 是先用卷积提取二维空间特征，最后分类前才 flatten
```

## 这节课的核心结论

MLP 看图片时，把图片当成一串 784 维数字。

CNN 看图片时，把图片当成有高度、宽度和通道的二维结构。

CNN 的三个关键直觉是：

```text
局部感受野
权重共享
feature map
```

这就是为什么 CNN 比普通 MLP 更适合图像任务。
